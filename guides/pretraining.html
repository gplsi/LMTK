Pretraining Guide#

Overview

This guide covers the pretraining capabilities of the ML Training Framework, focusing on distributed training strategies, optimization techniques, and configuration options.

Prerequisites

  • Basic understanding of transformer-based language models

  • Familiarity with PyTorch and distributed training concepts

  • Access to appropriate compute resources for model training

Pretraining Architecture#

The pretraining pipeline in the ML Training Framework is built around a modular architecture that supports different distributed training paradigms:

FSDP Configuration#
1from src.tasks.pretraining.fabric.wrappers.fsdp_config import get_fsdp_config
2
3fsdp_config = get_fsdp_config(
4    mixed_precision=True,
5    sharding_strategy="FULL_SHARD",
6    checkpoint_activation=True,
7    activation_checkpointing_policy="contiguous"
8)

Key Features:

  • Full parameter sharding across devices

  • Memory-efficient gradient computation

  • Support for checkpoint activation to reduce memory usage

  • Auto-wrapped module hierarchy

DDP Setup#
 1import torch.distributed as dist
 2from src.tasks.pretraining.fabric.distributed import setup_ddp
 3
 4setup_ddp(
 5    backend="nccl",
 6    find_unused_parameters=False
 7)
 8
 9# Model will be wrapped with DDP
10model = setup_model_with_ddp(model)

Key Features:

  • Efficient gradient synchronization

  • Lower communication overhead

  • Compatible with gradient accumulation

  • Suitable for homogeneous hardware setups

DeepSpeed Configuration#
 1# DeepSpeed configuration via JSON file
 2ds_config = {
 3    "train_batch_size": 32,
 4    "fp16": {
 5        "enabled": True
 6    },
 7    "zero_optimization": {
 8        "stage": 2,
 9        "offload_optimizer": {
10            "device": "cpu"
11        }
12    }
13}
14
15# Initialize with DeepSpeed
16model, optimizer, _, _ = deepspeed.initialize(
17    model=model,
18    config=ds_config
19)

Key Features:

  • ZeRO optimizer for memory efficiency

  • CPU offloading capability

  • Mixed precision training

  • Pipeline parallelism support

Training Orchestration#

The training process is managed by the Orchestrator class, which handles data loading, model initialization, training loops, and checkpointing:

        flowchart TD
   A[Configuration Loading] --> B[Model Initialization]
   B --> C[Optimizer Setup]
   C --> D[Data Preparation]
   D --> E[Training Loop]
   E --> F[Validation]
   F --> G{Continue?}
   G -->|Yes| E
   G -->|No| H[Save Model]
   E -->|Checkpoint| I[Save Checkpoint]
   I --> E

Performance Monitoring#

The framework includes tools for monitoring training performance:

 1from src.tasks.pretraining.fabric.speed_monitor import SpeedMonitor
 2
 3# Initialize the speed monitor
 4speed_monitor = SpeedMonitor(
 5    window_size=50,
 6    time_unit="hours"
 7)
 8
 9# Update during training
10for step in training_steps:
11    # Training logic
12    speed_monitor.update(samples=batch_size)
13
14    if step % log_interval == 0:
15        metrics = speed_monitor.compute()
16        logger.info(f"Training speed: {metrics['samples_per_second']:.2f} samples/second")

Warning

Monitoring overhead can impact training performance. Use appropriate logging intervals based on your training scale.

Complete Pretraining Example#
