Tokenization Guide#

Overview

This guide explains the tokenization capabilities of the ML Training Framework, including custom vocabulary building, specialized tokenizers for causal language models, and optimization techniques.

Key Concepts

  • Tokenization: The process of converting text into numerical token IDs

  • Vocabulary: A mapping between tokens and their numerical IDs

  • BPE (Byte-Pair Encoding): An algorithm for subword tokenization

  • WordPiece: An alternative subword tokenization algorithm

Tokenization Architecture#

The framework provides a modular tokenization system built around the BaseTokenizer class:

Tokenizer Class Hierarchy#
BaseTokenizer
├── CausalTokenizer
│   ├── GPT2Tokenizer
│   └── LlamaTokenizer
└── CustomTokenizer

Core Components#

Tokenizer Config
class TokenizerConfig:
    vocab_size: int
    min_frequency: int = 2
    special_tokens: List[str] = ["<unk>", "<pad>"]
    unk_token: str = "<unk>"
    pad_token: str = "<pad>"
    algorithm: str = "bpe"  # or "wordpiece"

Configuration for tokenizer training, including vocabulary size and special tokens.

Training Process
from src.tasks.tokenization.orchestrator import train_tokenizer

tokenizer = train_tokenizer(
    corpus_files=["data/corpus.txt"],
    config=config,
    output_dir="tokenizers/custom"
)

Process for training a tokenizer on a text corpus.

Causal Tokenization#

For causal language models, the framework offers specialized tokenizers that handle attention masking and positional information appropriately:

Using a Causal Tokenizer#
 1from src.tasks.tokenization.tokenizer.causal import CausalTokenizer
 2
 3tokenizer = CausalTokenizer.from_file("tokenizers/llama/tokenizer.json")
 4
 5# Tokenize with attention mask
 6encoded = tokenizer.encode(
 7    "Machine learning is fascinating",
 8    add_special_tokens=True,
 9    return_attention_mask=True
10)
11
12print(f"Token IDs: {encoded.ids}")
13print(f"Attention mask: {encoded.attention_mask}")

Configuration via YAML#

The tokenizer can be configured using YAML files for reproducible experiments:

Example Tokenizer Configuration#
 1tokenizer:
 2  vocab_size: 32000
 3  min_frequency: 3
 4  special_tokens:
 5    - "<unk>"
 6    - "<pad>"
 7    - "<s>"
 8    - "</s>"
 9  unk_token: "<unk>"
10  pad_token: "<pad>"
11  bos_token: "<s>"
12  eos_token: "</s>"
13  algorithm: "bpe"
14  normalization: "NFC"
15  add_prefix_space: true

Usage with Pretraining#

The tokenizer integrates seamlessly with the pretraining pipeline:

Integration with Pretraining#
 1from src.tasks.pretraining.orchestrator import PretrainingOrchestrator
 2from src.tasks.tokenization.tokenizer.causal import CausalTokenizer
 3from src.config.config_loader import load_config
 4
 5# Load tokenizer
 6tokenizer = CausalTokenizer.from_file("tokenizers/custom/tokenizer.json")
 7
 8# Load training config
 9config = load_config("config/experiments/llama-3.2-3b/continual.yaml")
10
11# Initialize orchestrator with tokenizer
12orchestrator = PretrainingOrchestrator(
13    config=config,
14    tokenizer=tokenizer
15)
16
17# Start training
18orchestrator.train()
